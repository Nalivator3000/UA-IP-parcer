# Где найти настройку автодеплоя в Railway

## Важно: Автодеплой может быть включен по умолчанию!

В новых версиях Railway автодеплой **включен автоматически** при подключении GitHub репозитория. Если вы видите:
- ✅ Source Repo: `Nalivator3000/UA-IP-parcer`
- ✅ Branch: `main`
- ✅ Root Directory: `/` (корень)

То автодеплой **уже работает**!

## Где проверить автодеплой:

### Вариант 1: В разделе Deployments

1. Откройте Railway Dashboard
2. Перейдите в раздел **"Deployments"** (вкладка вверху)
3. Проверьте последние деплои:
   - Если видите деплои после каждого push - автодеплой работает
   - Если деплои только ручные - автодеплой не работает

### Вариант 2: В настройках Service → Deploy

1. Railway Dashboard → Ваш проект
2. Settings → **Service** → вкладка **"Deploy"** (не "Settings"!)
3. Там должна быть настройка **"Auto Deploy"** или **"Watch"**

### Вариант 3: Через Railway CLI

```bash
# Установите Railway CLI
npm install -g @railway/cli

# Войдите
railway login

# Подключите проект
cd csv_export_app
railway link

# Проверьте настройки деплоя
railway status
```

## Как проверить, работает ли автодеплой:

1. **Сделайте тестовый коммит:**
   ```bash
   cd csv_export_app
   echo "# Test" >> test.txt
   git add test.txt
   git commit -m "Test autodeploy"
   git push origin main
   ```

2. **Проверьте Railway Dashboard:**
   - Откройте раздел **"Deployments"**
   - Должен появиться новый деплой автоматически
   - Если появился - автодеплой работает ✅
   - Если не появился - автодеплой не работает ❌

## Если автодеплой не работает:

### Решение 1: Переподключить репозиторий

1. Settings → Service → Source
2. Нажмите **"Disconnect"**
3. Затем **"Connect Repo"**
4. Выберите `Nalivator3000/UA-IP-parcer`
5. Выберите ветку `main`
6. Railway автоматически включит автодеплой

### Решение 2: Проверить webhook на GitHub

1. GitHub → Репозиторий `UA-IP-parcer` → Settings → Webhooks
2. Должен быть webhook от Railway
3. Проверьте последние доставки (Recent Deliveries)
4. Если webhook не работает - переподключите репозиторий в Railway

### Решение 3: Использовать ручной деплой

Если автодеплой не работает, можно использовать ручной деплой:
1. Railway Dashboard → Deployments
2. Нажмите **"Deploy"** или **"Redeploy"**
3. Это запустит деплой вручную

## Примечание:

В некоторых версиях Railway интерфейса настройка автодеплоя может называться:
- "Auto Deploy"
- "Watch"
- "Automatic Deployments"
- Или быть включена по умолчанию без отдельной настройки

Если вы видите, что деплои происходят автоматически после push - значит автодеплой работает, даже если нет явной настройки!
