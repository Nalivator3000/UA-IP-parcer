# Проверка автодеплоя в Railway (без явной настройки)

## ⚠️ Важно: В новых версиях Railway автодеплой включен по умолчанию!

Если вы видите в Settings:
- ✅ **Source Repo:** `Nalivator3000/UA-IP-parcer`
- ✅ **Branch:** `main` 
- ✅ **Root Directory:** `/` (корень)

То автодеплой **уже работает автоматически**! Отдельной настройки с галочкой может не быть.

## Как проверить, работает ли автодеплой:

### Способ 1: Проверка через Deployments

1. Откройте Railway Dashboard
2. Перейдите во вкладку **"Deployments"** (вверху, рядом с Settings)
3. Проверьте последние деплои:
   - **Если видите деплои после каждого `git push`** → автодеплой работает ✅
   - **Если деплои только ручные** → автодеплой не работает ❌

### Способ 2: Тестовый push

1. Сделайте небольшое изменение:
   ```bash
   cd csv_export_app
   echo "# Test" >> README.md
   git add README.md
   git commit -m "Test autodeploy"
   git push origin main
   ```

2. Откройте Railway Dashboard → **Deployments**
3. Должен появиться новый деплой автоматически (через 1-2 минуты)
4. Если появился → автодеплой работает ✅
5. Если не появился → автодеплой не работает ❌

### Способ 3: Проверка webhook на GitHub

1. Откройте репозиторий на GitHub: https://github.com/Nalivator3000/UA-IP-parcer
2. Перейдите в **Settings** → **Webhooks**
3. Должен быть webhook от Railway
4. Откройте webhook → **Recent Deliveries**
5. Проверьте последние доставки:
   - Если есть успешные доставки после push → автодеплой работает ✅
   - Если нет доставок или ошибки → автодеплой не работает ❌

## Если автодеплой не работает:

### Решение 1: Переподключить репозиторий

1. Railway Dashboard → Settings → Service → **Source**
2. Нажмите **"Disconnect"**
3. Затем **"Connect Repo"**
4. Выберите `Nalivator3000/UA-IP-parcer`
5. Выберите ветку `main`
6. Railway автоматически настроит автодеплой

### Решение 2: Проверить webhook

1. GitHub → Settings → Webhooks
2. Если webhook отсутствует или не работает:
   - Переподключите репозиторий в Railway
   - Или создайте webhook вручную (но лучше переподключить)

### Решение 3: Использовать ручной деплой

Если автодеплой не работает, можно деплоить вручную:
1. Railway Dashboard → **Deployments**
2. Нажмите **"Deploy"** или **"Redeploy"**
3. Это запустит деплой вручную

## Примечание:

В некоторых версиях Railway интерфейса настройка автодеплоя может:
- Быть включена по умолчанию без отдельной настройки
- Находиться в разделе **"Deploy"** (не "Settings")
- Называться по-другому (например, "Watch" или "Automatic Deployments")

**Главное:** Если деплои происходят автоматически после push - значит автодеплой работает, даже если нет явной настройки!
