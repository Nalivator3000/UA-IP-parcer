# Исправление ошибки "GitHub Repo not found" в Railway

## Проблема

Railway показывает ошибку:
```
GitHub Repo not found
```

И не может сделать редеплой из-за потери связи с репозиторием GitHub.

## Решение: Переподключить репозиторий

### Шаг 1: Проверьте имя репозитория

**Важно:** Проверьте точное имя репозитория на GitHub:
- `UA-IP-parcer` (с "c") 
- или `UA-IP-parser` (с "s")?

Откройте: https://github.com/Nalivator3000/UA-IP-parcer

### Шаг 2: Отключите текущее подключение

1. **Railway Dashboard** → проект "UA-IP-parcer" → **Settings**
2. Найдите секцию **"Source Repo"**
3. Нажмите кнопку **"Disconnect"**
4. Подтвердите отключение

### Шаг 3: Подключите репозиторий заново

1. В той же секции **"Source Repo"** нажмите **"Connect Repo"** или **"New"** → **"GitHub Repo"**
2. Выберите репозиторий: **`Nalivator3000/UA-IP-parcer`**
3. Railway автоматически подключит репозиторий

### Шаг 4: Проверьте настройки

После подключения проверьте:

1. **Branch connected to production:**
   - Должна быть выбрана ветка `main` (или `master`)
   - Ошибка "GitHub Repo not found" должна исчезнуть

2. **Root Directory:**
   - Оставьте **ПУСТЫМ** (все файлы в корне репозитория)

3. **Auto Deploy:**
   - Убедитесь, что включен автоматический деплой

### Шаг 5: Проверьте доступ Railway к GitHub

1. **GitHub** → **Settings** → **Applications** → **Authorized OAuth Apps**
2. Найдите **"Railway"**
3. Если его нет или доступ отозван:
   - При подключении репозитория Railway запросит авторизацию
   - Разрешите доступ к репозиторию

## Альтернативное решение: Создать новый проект

Если переподключение не помогает:

1. **Railway Dashboard** → **New Project**
2. Выберите **"Deploy from GitHub repo"**
3. Выберите репозиторий `Nalivator3000/UA-IP-parcer`
4. Railway автоматически определит Node.js проект
5. Добавьте переменные окружения:
   - `DATABASE_URL` - строка подключения к PostgreSQL
   - `PORT` - Railway установит автоматически
   - `NODE_ENV` - `production`

6. Удалите старый проект (если нужно)

2. **Проверьте Railway:**
   - Должен появиться новый деплой автоматически
   - В логах должно быть видно процесс деплоя
   - Ошибка "GitHub Repo not found" должна исчезнуть

## Частые причины проблемы

1. **Репозиторий переименован** - проверьте точное имя на GitHub
2. **Репозиторий стал приватным** - Railway должен иметь доступ
3. **Railway потерял авторизацию** - переподключите через OAuth
4. **Неправильное имя репозитория** - проверьте регистр и спецсимволы

1. **Проверьте логи Railway:**

2. **Проверьте настройки GitHub:**

   - Railway должен быть авторизован

3. **Создайте новый проект** с нуля и перенесите переменные окружения
