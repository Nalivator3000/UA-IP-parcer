# Настройка Root Directory в Railway для UA-IP-parcer

## Структура проекта

Все файлы приложения находятся в **корне репозитория**:

```
UA-IP-parcer/
├── server.js          ← главный файл приложения
├── package.json       ← зависимости Node.js
├── public/            ← статические файлы (HTML, CSS, JS)
│   └── index.html
├── railway.json       ← конфигурация Railway
└── ...
```

## Настройка Root Directory

### ✅ Правильная настройка:

**Root Directory:** оставьте **ПУСТЫМ** или укажите `.` (точка)

Railway будет искать файлы в корне репозитория, где и находятся все необходимые файлы.

### ❌ НЕ указывайте:

- `/` - может вызвать проблемы
- `./` - избыточно, но работает
- Любую поддиректорию - файлы находятся в корне

## Проверка

После настройки Railway должен:

1. ✅ Найти `package.json` в корне
2. ✅ Найти `server.js` в корне  
3. ✅ Найти `railway.json` в корне
4. ✅ Успешно выполнить `npm install`
5. ✅ Запустить `npm start` (который запускает `node server.js`)

## Если Root Directory уже указан неправильно

1. Откройте Railway Dashboard → проект UA-IP-parcer → Settings
2. Найдите секцию "Source" → "Add Root Directory"
3. **Удалите** текущее значение (оставьте пустым)
4. Или укажите `.` (точка)
5. Сохраните изменения
6. Railway автоматически пересоберет проект

## Важно

- **superset-railway** и **UA-IP-parcer** - это **разные репозитории**
- Они находятся в разных директориях локально
- Они не должны перемешиваться
- Каждый проект имеет свой собственный репозиторий на GitHub
